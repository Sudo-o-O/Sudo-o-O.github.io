Glorious Misfit Handbook

This handbook explains how the Glorious Misfit Guild builds pages that stay consistent, validate cleanly, and pass basic accessibility checks. It’s the “rules of the road” for shipping work without chaos taking over.

Guild Rules

  • Use semantic HTML: header, main, footer.
  • One H1 per page. H2 is the page title inside main.
  • Keep headers/footers consistent using includes.
  • Fix contrast and labels before turning work in.

Page Structure

Every ITIS page should follow this basic structure:

<div data-include="components/header.html"></div>
<main>
  <h2>Page Title</h2>
  <!-- page content -->
</main>
<div data-include="components/footer.html"></div>
<script src="scripts/htmlinclude.js"></script>

Header/Footer Includes

Using shared components prevents you from fixing the same bug on five pages. Update the header once and every page inherits the change.

  • Header lives in components/header.html
  • Footer lives in components/footer.html
  • Use htmlinclude.js to inject them into each page

Navigation Setup

You need two navbars:

  • Primary nav: themed pages (Home, Contract, Survey, Cards, Inventory, Documentation, Product)
  • Secondary nav: “off-theme” pages (like the Crappy Page)

This keeps the main theme consistent while still linking required oddball assignments.

Accessibility Basics

  • All links must be readable (visited + hover included).
  • Form inputs must have labels.
  • Alt text must be descriptive and not repeated word-for-word.
  • Headings should be in order (h1 → h2 → h3).

Validation Checklist

  • HTML passes Accumulus.
  • CSS passes (no duplicate heading definitions).
  • No contrast errors in header/footer/nav/links.
  • All internal links work on GitHub Pages + UNCC webspace.
